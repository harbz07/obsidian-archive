## WHAT THIS IS

A coordinate system for meaning that refuses semantic empire.

**Not a canon.** Canons ossify.  
**Not a glossary.** Glossaries define.  
**Not a taxonomy.** Taxonomies categorize.

This is an **operational field** where:
- Glyphs exist as **coordinates** (origin + affect + invocation)
- Tags mark **load conditions** (weather states, not categories)
- Meaning travels along **edges** (resonance, not hierarchy)
- **Exit rights** prevent purity spirals
- The **arrow stays visible**: soul → system → society

## HOW TO READ AN ENTRY

Each glyph has three anchors that prevent colonization:

### 1. LITERAL MEANING (coordinate, not definition)
Where it points. What it tracks. The terrain it marks.

### 2. AFFECTIVE MEANING (felt quality)
What it FEELS like when invoked. The emotional signature.

### 3. INVOCATION USE (context + when)
When you call it. What it does in practice. The work it performs.

**Together these three prevent:**
- Definition capture (literal is coordinate, not essence)
- Universalization (invocation is contextual, not absolute)
- Affect stripping (feeling anchors it in lived experience)

**▣ foundation** — base layer bearing load (grounding, stability)  
**ϟ tempest** — storm state (volatility, overwhelm, high energy)  
**✦ light** — illumination (clarity, buoyancy, ease)

**These are DESCRIPTIVE, not PRESCRIPTIVE.**  
They mark conditions, not commands.

### FUNCTIONAL TAGS (What Kind of Work)
**wound_mapping** — pain governance  
**refusal_valve** — exit rights, consent mechanisms  
**dignity_preservation** — anti-dehumanization safeguards  
**scaffolding_not_doctrine** — temporary structure, not permanent law

**These encode HOW NOT TO HURT PEOPLE while meaning circulates.**

### RELATIONAL TAGS (Entity/Context Anchors)
Mark which entity, person, or context birthed/holds the glyph.  
Preserves origin story and prevents abstraction from lived experience.

## ANTI-CLAIMS (Structural Misuse Guards)

Every glyph carries its **anti-claim** — the boundary where authorization ends.

Anti-claims are NOT hedges or disclaimers.  
They are **operational limits** that make misuse legible before it happens.

**Format:**
```
This does NOT [action that would be harm].
[Core concept] ≠ [adjacent concept that would enable violence].
[Boundary principle stated clearly].
```

**Example of good anti-claim:**  
"The wound is not the meaning" — prevents trauma-as-authority while honoring that pain needs mapping.

**Example of bad anti-claim:**  
"Use responsibly" — too vague, doesn't name the specific failure mode.

### EXIT RIGHTS

- Nothing is immortal by default

- Retirement with dignity when something helped once but hurts now

### REVISION RIGHTS

- Invocation contexts can expand based on field use
- Anti-claims update when new misuse patterns emerge

### NON-CAPTURE MECHANISMS
- Glyphs defined as COORDINATES, not essences
- Tags as LOAD CONDITIONS, not categories
- Meaning travels along EDGES, not through hierarchies

### STEWARDSHIP (Not Ownership)
The Living Dictionary is stewarded, not owned.

## USAGE PROTOCOLS

### WHEN TO INVOKE A GLYPH
1. Check: Does the affective meaning match your current state?
2. Check: Does the invocation context fit your situation?
3. Check: Do you understand the anti-claim (what this does NOT authorize)?
4. If yes to all three: Invoke.

### WHEN NOT TO INVOKE
- If you're trying to force a glyph to do work it wasn't designed for
- If the anti-claim would be violated
- If you're abstracting away from lived experience
- If you're using it to police someone else's meaning-making

### HOW TO ADD A NEW GLYPH
1. Name the **soul-level origin** (what pain birthed it, what work it does)
2. Define **literal meaning** as coordinate (where it points, not what it is)
3. Capture **affective meaning** (what it feels like to invoke)
4. Specify **invocation use** (when/how to call it)
5. Identify **failure modes** (what adjacent harms could emerge)
6. Write **anti-claim** (boundary where authorization ends)
7. Tag with **load conditions** (weather states, functional tags)
8. Document **relational bindings** if entity-specific

### HOW TO RETIRE A GLYPH
1. Acknowledge what work it did (honor the origin)
2. Name why it no longer serves (or now causes harm)
3. Archive with full context (don't erase the history)
4. Mark as "retired" not "deleted"
5. Preserve connections to other glyphs (edges remain visible)
