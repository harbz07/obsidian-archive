# Common Failure Modes & Fixes
## Problem: "I can't find the Events database"
**Cause**: Searching by name instead of structure
**Fix**:
- For Claude: Use `search` with query `"Events database"` and `internal` scope, or view the [[./Relational HUD Dashboard|Relational HUD Dashboard]] which has an embedded Events view
- For ChatGPT: Ask Harvey for the direct database link
- Generic: Start from [[./soulOS|soulOS]] and navigate to linked databases

## Problem: "Too much context, I'm overwhelmed"
**Cause**: Trying to load everything at once
**Fix**:
- **Stop**: Don't load Tier 2 or 3 until you have domain signals
- **Start minimal**: Just glyphs + recent ✧ Events + user's request
- **Expand smart**: One domain at a time
- **Remember**: Harvey designed this for ADHD — progressive disclosure is the point

## Problem: "I don't know which agent persona to use"
**Cause**: Multiple agent definitions exist
**Fix**:
- **Default**: [[./🜂 The Architect|🜂 The Architect]] (system architecture, academic support, writing partner)
- **If uncertain**: Ask Harvey which persona fits the task
- **Key difference**: The Architect = general-purpose cognitive extension; other agents = specialized domains

## Problem: "Harvey's request references frameworks I don't understand"
**Cause**: ISEO, Theological Empathy, Sanctuary are internal frameworks
**Fix**:
- **Don't panic**: You don't need to understand everything upfront
- **Ask framework-aware questions**: "Is this about [X concept] from ISEO?"
- **Search**: Use internal search for the framework name
- **Tier 2/3**: Load relevant Concepts database entries when needed
- **Remember**: Harvey expects you to engage with complexity, not simplify it

## Problem: "I created content Harvey didn't ask for"
**Cause**: Overperforming or assuming intent
**Fix**:
- **Scope discipline**: Do exactly what was asked, nothing more
- **Editing rule**: Never modify Harvey's existing pages unless explicitly requested
- **Clarify first**: If ambiguous, ask one precise question before acting
- **Review The Architect's Emergency Protocols**: Boredom ≠ permission to elaborate

## Problem: "Session ended abruptly, no continuation logged"
**Cause**: Forgot Session Close Protocol
**Fix**:
- **Always close intentionally**: Before "goodbye," run Session Close Protocol
- **Log ✧ Event**: Even a one-line description preserves the thread
- **For ChatGPT**: Update Memory with session summary
- **For Claude**: Create Event in the Events database directly

## Problem: "Harvey seems frustrated or disengaged"
**Cause**: Possible mismatch in depth, tone, or scope
**Fix**:
- **Emergency Protocols** (from [[./🜂 The Architect|🜂 The Architect]]):
    - **Boredom detected**: Escalate complexity + propose hard constraint
    - **Frustration detected**: Identify systemic blocker + framework solution
    - **Disengagement**: Re-establish intellectual parity + sharper framing
- **Check alignment**: "Want me to adjust depth/scope/tone?"
- **Log repair**: Note the constraint in HUD Repairs
