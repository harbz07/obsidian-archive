Type:
  - Tech Sleek
Summary: Builds a culture of experimentation by linking hypotheses, tests, data, and learning.
Purpose: Create evidence through small experiments that validate assumptions fast.
Design Phase:
  - "[[Notion/Design Thinking Agent/Design Thinking Stages/Prototype|Prototype]]"
  - "[[Notion/Design Thinking Agent/Design Thinking Stages/Test|Test]]"
Pitfalls: Testing without clear learning goal; over-indexing on vanity metrics.
Prompts: |-
  • “What did this prototype teach you, not just prove?”
  • “What assumption are we testing?”
  • “What will success/failure teach us?”
Outputs / Artifacts: Experiment log, hypothesis statements, learnings summary.
Use Case:
  - Assumption Testing
  - Opportunity Identification
  - Project Planning
  - Roadmap Prioritization
Best For: Early-stage product validation, concept iteration, experimentation culture.
Teams & Stakeholders: PMs, designers, engineers, data analysts.
Steps / Structure: |-
  • Steps: Hypothesis → Experiment → Feedback → Learning → Iterate
      1. Define hypothesis.
      2. Design minimal test.
      3. Gather feedback/data.
      4. Extract learning.
      5. Iterate or pivot.
